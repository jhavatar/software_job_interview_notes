Programming Paradigms

Programming Paradigms

fundamental style of computer programming

Imperative Programming

  • Computation as statements that directly change a program state
  • characterized by:
    • Direct assignments
    • common data structures
    • global variables
    • goto statements
  • Examples: C, C++, Java, PHP, Python

Structured Programming

  • A style of imperative programming with more logical program structure
  • Aimed on improving the clarity, quality, and development time of a computer program
  • characterized by:
    • indentation
    • either no, or limited use of, goto statements
  • Examples: C, C++, Java

Procedural Programming

  • Derived from structured programming
  • based on the concept of modular programming or the procedure call
  • characterized by:
    • Local variables
    • sequence
    • selection
    • iteration
    • modularization
  • Examples: C, C++, Java, PHP, Python

Functional Programming

  • Treats computation as the evaluation of mathematical functions
  • avoiding state and mutable data
  • characterized by:
    • Lambda calculus
    • compositionality -- the meaning of a complex expression is determined by the meanings of its constituent expressions and the rules used to combine them
    • formula
    • recursion
    • referential transparency -- an expression can be replaced with its value without changing the behavior of a program
    • no side effects -- does not: in addition to returning a value, also modifies some state
  • Examples: Erlang, Haskell, Lisp, Clojure, Scala

Event-driven Programming

  • Program flow is determined mainly by events, such as mouse clicks or interrupts including timer
  • characterized by:
    • Main loop
    • event handlers
    • asynchronous processes
  • Examples: ActionScript

Object-oriented Programming

  • Treats datafields as objects manipulated through pre-defined methods only

  • Examples: C++, C#, Java, PHP, Python, Ruby, Scala

Declarative Programming

  • Defines computation logic without defining its detailed control flow
  • attempt to minimize or eliminate side effects by describing what the program should accomplish in terms of the problem domain, rather than describing how to go about accomplishing it as a sequence of the programming language primitives
  • characterized by:
    • Fourth-generation programming language (4GLs)
    • spreadsheets
    • report program generators
  • Examples: SQL, regular expressions, CSS

Automata-based programming

  • Treats programs as a model of a finite state machine or any other formal automata
  • characterized by:
    • State enumeration
    • control
    • state changes
    • isomorphism
    • state transition table
