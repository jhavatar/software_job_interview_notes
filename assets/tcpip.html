Internet protocol suite

Internet Protocol Suite

  • aka TCP/IP
  • Is the networking model and a set of communications protocols used for the Internet and similar networks.

Terminology

Packet-switched Network

  • A digital communications network that groups all transmitted data, irrespective of content, type, or structure into suitably sized blocks, called packets.
  • Each packet has to find its own route to the destination.
  • No predetermined path; the decision as to which node to hop to in the next step is taken only when a node is reached.
  • pros:
    • Optimize utilization of available link capacity, minimize response times and increase the robustness of communication.
    • cheaper -- share a network
  • cons:
    • variable delay and throughput depending on the traffic load in the network.

Circuit switched Network

  • Circuit switching is a methodology of implementing a telecommunications network in which two network nodes establish a dedicated communications channel (circuit) through the network before the nodes may communicate.
  • pro:
    • the full bandwidth of the channel and remains connected for the duration of the communication session.
    • no overhead with packets
  • cons:
    • unused capacity guaranteed to a connection cannot be used by other connections on the same network

Datagram

Basic transfer unit associated with a packet-switched network in which the delivery, arrival time, and order of arrival are not guaranteed by the network service.

Routing

Process of selecting paths in a network along which to send network traffic

Bridging

  • network equipment (bridges) allow two or more communication networks, or two or more network segments, to create an aggregate network.

Network switch

A network switch or switching hub is a computer networking device that links network segments or network devices

Local Area Network (LAN)

Computer network that interconnects computers in a limited area such as a home, school, computer laboratory, or office building using network media.

Wide Area Network

Network that covers a broad area using private or public network transports.

Internet

Specific global system of interconnected networks that use the Internet protocol suite

World Wide Web (WWW)

Global system of interlinked hypertext documents, images and other resources, logically interrelated by hyperlinks and referenced with Uniform Resource Identifiers (URIs).

Uniform Resource Identifier (URI)

  • String of characters used to identify a name or a web resource
  • Can be URL or URN
    • A uniform resource name (URN) defines an item's identity, like a person's name
    • uniform resource locator (URL) provides a method for finding an item, like a person's street address:

Hypertext

  • a structured text displayed on an electronic devices that contain logical links/references (hyperlinks) to other text (or tables or images) which the reader can immediately access.
  • underlying concept defining the structure of the World Wide Web

Domain Name System

  • a hierarchical distributed naming system for computers, services, or any resource connected to the Internet or a private network.
  • translates easily memorized domain names to the numerical IP addresses needed for the purpose of locating computer services and devices worldwide

Internet protocol suite Layers

  • Internet protocol has four abstraction layers, sorts all related protocols according to the scope of networking involved.
  • layers lowest to hightest:
    1. link layer -- communication technologies for a single network segment (link) of a local area network
    2. internet layer -- connects independent networks, thus establishing internetworking.
    3. transport layer -- handles host-to-host communication
    4. application layer -- protocols for specific data communications services on a process-to-process level.

Link layer

  • lowest component layer
  • move packets between the Internet layer interfaces of two different hosts on the same link
  • e.g. software device drivers for the network card, firmware, specialized chipsets

Internet layer

  • Routing packets across potentially multiple networks, from source network to destination network
  • Agnostic of the data structures and protocols of the Transport layer
  • e.g. Internet Protocol (IP)

Transport layer

  • Establishes a basic data channel that an application uses in its task-specific data exchange.
  • establishes host-to-host connectivity independent of the structure of user data
  • e.g. Transmission Control Protocol (TCP), User Datagram Protocol (UDP)

Application layer

  • higher-level protocols used by most applications for network communication
  • e.g. Hypertext Transfer Protocol (HTTP), File Transfer Protocol (FTP), the Simple Mail Transfer Protocol (SMTP)

Internet Protocol (IP)

  • primary protocol in the Internet layer of the Internet protocol suite
  • tasks:
    • Host addressing and identification: This is accomplished with a hierarchical IP addressing system.
    • Packet routing: delivering packets from the source host to the destination host across multiple potential networks solely based on the IP addresses in the packet headers.

Datagram construction

  • has two components:
    • IP header -- header is tagged with the source IP address, the destination IP address, and other meta-data needed to route and deliver the datagram.
    • payload -- data that is transported.

IP addressing and routing

  • IP addressing: assignment of IP addresses and associated parameters to host interfaces.
  • address space is divided into networks and subnetworks, involving the designation of network or routing prefixes.
  • IP routing is performed by all hosts, but most importantly by routers, which transport packets across network boundaries.

Data Transfer

  • unreliable -- best effort delivery, no guarantees that data is delivere

Transmission Control Protocol (TCP)

  • One of the core protocols of Transport layer of the Internet protocol suite

Connection establishment

  • Before a client attempts to connect with a server, the server must first bind to and listen at a port to open it up for connections: this is called a passive open.
  • Once the passive open is established, can open connection with three-way handshake:
    1. SYN -- active open is performed by the client sending a SYN to the server. The client sets the segment's sequence number to a random value A.
    2. SYN-ACK -- In response, the server replies with a SYN-ACK. The acknowledgment number is set to one more than the received sequence number i.e. A+1, and the sequence number that the server chooses for the packet is another random number, B.
    3. ACK -- Finally, the client sends an ACK back to the server. The sequence number is set to the received acknowledgement value i.e. A+1, and the acknowledgement number is set to one more than the received sequence number i.e. B+1.

Connection termination

  • Uses a four-way handshake, with each side of the connection terminating independently
  • endpoint wishing to stop its half of the connection sends FIN packet and other side responmds with ACK package

Data transfer

  • Reliable -- manages message acknowledgment, retransmission and timeout.
  • Ordered data transfer — the destination host rearranges according to sequence number
  • Heavyweight -- requires three packets to set up a socket connection, before any user data can be sent.
  • Flow control — limits the rate a sender transfers data to guarantee reliable delivery. e.g. Window scaling -- change receiving window size, i.e. change amount of data can receive before have to acknowledge sender
  • Congestion control

Aplications

  • point of sale
  • accounting
  • database systems

User Datagram Protocol (UDP)

  • One of the core protocols of Transport layer of the Internet protocol suite (IP)

Data transfer

  • Unreliable -- There is no handshaking dialogues, concept of acknowledgment, retransmission, or timeout.
  • Not ordered – If two messages are sent to the same recipient, the order in which they arrive cannot be predicted.
  • lightweight – There is no ordering of messages, no tracking connections, etc. It is a small transport layer designed on top of IP.
  • Datagrams:
    • only checked for integrity only if they arrive
    • provides checksums for data integrity
  • No congestion control

Aplications

  • Domain Name System (DNS), Simple Network Management Protocol (SNMP), the Routing Information Protocol (RIP)[2] and the Dynamic Host Configuration Protocol (DHCP)
  • Voice and video traffic
  • Realtime streaming
  • online games

Hypertext Transfer Protocol (HTTP)

  • One of the protocols of Application layer of the Internet protocol suite
  • foundation of data communication for the World Wide Web
  • the protocol to exchange or transfer hypertext.
  • functions as a request-response protocol in the client-server computing model. e.g. web browser as client and web site as server
  • HTTP resources are identified and located on the network by Uniform Resource Identifiers (URIs)
  • HTTP is a stateless protocol. Does not require server to retain information anout each user

HTTP Session

  • HTTP session is a sequence of network request-response transactions.
  • An HTTP client initiates a request by establishing a Transmission Control Protocol (TCP) connection to a particular port on a server (typically port 80)
  • An HTTP server listening on that port waits for a client's request message. Upon receiving the request, the server sends back a status line (e.g. "HTTP/1.1 200 OK") and a message of its own.
  • The body of this message is typically the requested resource, although an error message or other information may also be returned.

Request Methods

  • HTTP defines methods (sometimes referred to as verbs) to indicate the desired action to be performed on the identified resource.
  • GET: Requests a representation of the specified resource. should only retrieve data and should have no other effect.
  • HEAD: Asks for the response identical to the one that would correspond to a GET request, but without the response body
  • POST: Requests that the server accept the entity enclosed in the request as a new subordinate of the web resource identified by the URI. The data POSTed might be: annotation for existing resources; a message for a bulletin board, a block of data that is the result of submitting a web form to a data-handling process; or an item to add to a database.
  • PUT: Requests that the enclosed entity be stored under the supplied URI. If the URI exists, modify resource; else, server can create the resource with that URI.
  • DELETE: Deletes the specified resource.
  • TRACE: Echoes back the received request so that a client can see what (if any) changes or additions have been made by intermediate servers.
  • OPTIONS: Returns the HTTP methods that the server supports for specified URL.
  • CONNECT: Converts the request connection to a transparent TCP/IP tunnel, usually to facilitate SSL-encrypted communication (HTTPS) through an unencrypted HTTP proxy
  • PATCH: Is used to apply partial modifications to a resource

Steps when clicking web page link

Assuming the simplest possible HTTP request, no proxies and IPv4

  1. browser checks cache; if requested object is in cache and is fresh, skip to #9
  2. browser asks OS for server's IP address
  3. OS makes a DNS lookup and replies the IP address to the browser
  4. browser opens a TCP connection to server (this step is much more complex with HTTPS)
  5. browser sends the HTTP request through TCP connection
  6. browser receives HTTP response and may close the TCP connection, or reuse it for another request
  7. browser checks if the response is a redirect (3xx result status codes), authorization request (401), error (4xx and 5xx), etc.; these are handled differently from normal responses (2xx)
  8. if cacheable, response is stored in caches
  9. browser decodes response (e.g. if it's gzipped)
  10. browser determines what to do with response (e.g. is it a HTML page, is it an image, is it a sound clip?)
  11. browser renders response, or offers a download dialog for unrecognized types
